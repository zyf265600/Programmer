# Go 语言：append 与 copy 核心总结

## 1. 定义与核心语义

### **append**

- **定义**：内建函数，用于向 slice 末尾追加元素。
- **语法**：`s = append(s, elems...T)`
- **核心语义**：将元素接到 slice 尾部，并返回一个新的 **slice header**。
- **注意**：必须接住返回值（`s = append(...)`），因为 append 可能会触发扩容导致底层数组地址、长度或容量改变。

### **copy**

- **定义**：内建函数，用于将元素从源 slice (`src`) 复制到目标 slice (`dst`)。
- **语法**：`n := copy(dst, src)`
- **核心语义**：**按元素进行值复制**（element-wise value copy），返回实际复制的个数：$min(len(dst), len(src))$。
- **特性**：天然支持内存重叠（Overlap）的正确处理。

------

## 2. 底层机制与 Slice 结构

Slice 本质上是一个 **Header**：

```go
type slice struct {
    ptr *T   // 指向底层数组
    len int  // 当前长度
    cap int  // 当前容量
}
```

### append 的两种情况

1. **容量充足**：直接在底层数组 `ptr+len` 的位置写入新值，返回 `len+1` 的新 header。复杂度：时间 $O(1)$，空间 $O(1)$。
2. **容量不足**：触发 `growslice`。分配更大的新数组 $\rightarrow$ 拷贝旧数据 $\rightarrow$ 追加新数据 $\rightarrow$ 返回新 header。复杂度：时间 $O(n+k)$，空间 $O(n+k)$。
   - **均摊复杂度**：多次连续 append 的均摊时间复杂度为 $O(1)$。

------

## 3. 深拷贝 vs 浅拷贝的真相

Go 中没有绝对的“深/浅拷贝”函数，`append` 和 `copy` 遵循同样的规则：**按元素值复制**。最终效果取决于 **元素类型 (`T`)**：

| **元素类型**                    | **复制内容**    | **效果感官** | **备注**                    |
| ------------------------------- | --------------- | ------------ | --------------------------- |
| **基本类型** (int, byte, etc.)  | 数值本身        | **深拷贝**   | 修改新 slice 不影响原 slice |
| **纯值结构体** (struct)         | 整个结构体的值  | **深拷贝**   | 内部不含指针/引用           |
| **指针** (*T)                   | 指针地址        | **浅拷贝**   | 两个 slice 指向同一个对象   |
| **引用类型** (slice, map, chan) | Header/内部引用 | **浅拷贝**   | 内层底层数据仍是共享的      |

------

## 4. 内存重叠 (Overlap) 处理

- **copy**：专门处理重叠情况。例如 `copy(s[1:], s)` 是安全的，其逻辑类似于 C 语言中的 `memmove`。
- **append**：并非为重叠设计，但在删除元素的标准写法 `append(s[:i], s[i+1:]...)` 中利用了底层数组共享（虽然是两个slice header）的特性，是安全且高效的。

------

## 5. 复杂度总表

| **操作**            | **时间复杂度**                           | **额外空间复杂度**                     |
| ------------------- | ---------------------------------------- | -------------------------------------- |
| `append(s, x)`      | 均摊 $O(1)$，最坏 $O(n)$                 | $O(1)$ (不扩容) / $O(new\_cap)$ (扩容) |
| `append(s, t...)`   | $O(len(t))$，若扩容则 $O(len(s)+len(t))$ | 同上                                   |
| `copy(dst, src)`    | $O(min(len(dst), len(src)))$             | $O(1)$                                 |
| **Clone Slice**     | $O(n)$                                   | $O(n)$                                 |
| **删除第 i 个元素** | $O(n-i)$                                 | $O(1)$ (通常不扩容)                    |
| **插入第 i 个元素** | 最坏 $O(n)$                              | 若触发扩容则 $O(n)$                    |

------

## 6. 常用操作模板

### 6.1 克隆 (Clone)

```go
// 推荐：语义明确
b := make([]int, len(a))
copy(b, a)

// 另一种写法
b := append([]int{}, a...)
```

### 6.2 删除第 i 个元素 (保持顺序)

```
s = append(s[:i], s[i+1:]...)
```

### 6.3 插入元素到第 i 位 (原地思路)

```
s = append(s, 0)          // 1. 扩容/增加长度
copy(s[i+1:], s[i:])      // 2. 元素后移
s[i] = x                  // 3. 插入新值
```

### 6.4 头部删除 (Pop Front)

```
x, s := s[0], s[1:]       // 注意：底层数组前部的空间不会立即释放
```

------

## 7. 避坑指南

1. **append 丢失更新**：切记 `s = append(s, ...)`，不要漏掉赋值。
2. **copy 目标长度**：**==`copy` 不会自动扩容 `dst`。==**如果 `dst` 是 `nil` 或长度为 0，将什么都不会复制。
   - 错误：`var d []int; copy(d, s)`
   - 正确：`d := make([]int, len(s)); copy(d, s)`
3. **大数组残留**：在长生命周期的 slice 上进行头部删除或切片操作（比如只想留一小部分），在这之后由于底层长数组依然被引用，可能导致内存无法及时回收。必要时请使用 `copy` 迁移到新 slice 以释放旧长数组。
