# Go Program initialization and execution

程序初始化与执行

零值

当为变量分配存储空间时，无论是通过声明、调用 new、创建复合字面量，或调用 make，如果没有提供显式初始化，则变量会被赋予默认值。

该默认值称为该类型的零值：

布尔类型为 false
数值类型为 0
字符串为 ""
指针、函数、接口、切片、通道、映射为 nil

这种初始化是递归进行的。例如，若数组元素是结构体，则其字段也会被初始化为零值。

以下两个声明是等价的：

```go
var i int
var i int = 0
```

在执行：

```go
type T struct { i int; f float64; next *T }
t := new(T)
```

之后：

```go
t.i == 0
t.f == 0.0
t.next == nil
```

以下声明结果相同：

```go
var t T
```

包初始化

在一个包内部，包级变量初始化是逐步进行的。每一步选择声明顺序中最早、且不依赖未初始化变量的变量进行初始化。

更精确地说，如果一个包级变量尚未初始化，并且：

没有初始化表达式，或
其初始化表达式不依赖未初始化变量

则该变量被认为已准备好初始化。

初始化过程会重复执行，按声明顺序选择下一个已准备好的变量，直到没有可初始化变量为止。

如果结束时仍有未初始化变量，则存在初始化循环，程序无效。

当多个变量通过单个多值表达式初始化时，它们会在同一步骤中一起初始化。

```go
var x = a
var a, b = f()
```

a 和 b 会在 x 之前一起初始化。

空变量在初始化过程中与普通变量相同。

若变量分布在多个文件中，其声明顺序由编译器接收文件的顺序决定。为确保可复现行为，建议构建系统按文件名字典序提供文件。

依赖分析只基于源码中的词法引用，而非变量的实际值。

例如：

```go
var (
	a = c + b
	b = f()
	c = f()
	d = 3
)

func f() int {
	d++
	return d
}
```

初始化顺序为：d, b, c, a。

初始化表达式中子表达式顺序不影响整体初始化顺序。

依赖分析只在当前包内进行。若存在隐藏的数据依赖，则初始化顺序未指定。

例如：

```go
var x = I(T{}).ab()
var _ = sideEffect()
var a = b
var b = 42

type I interface { ab() []int }
type T struct{}
func (T) ab() []int { return []int{a, b} }
```

a 会在 b 之后初始化，但 x 的初始化时机未指定，因此 sideEffect 的调用时机也未指定。

变量也可以通过名为 init 的函数初始化。

```go
func init() { … }
```

一个包可以定义多个 init 函数。

init 只能用于声明初始化函数，不能被引用。

整个包初始化过程为：

先为所有包级变量赋初值
然后按源码顺序调用所有 init 函数

程序初始化

完整程序的所有包按步骤初始化。若一个包有导入，则先初始化其导入的包。每个包只初始化一次。导入机制保证不存在循环依赖。

具体过程：

将所有包按导入路径排序
每一步选择第一个未初始化且其导入包已初始化的包
重复直到全部完成

包初始化在单个协程中顺序执行。init 函数可以启动其他协程，但初始化过程不会并发执行多个 init 函数。

程序执行

完整程序通过链接一个未被导入的 main 包以及其所有依赖包构成。

main 包必须：

包名为 main
声明函数 main
函数无参数且无返回值

```go
func main() { … }
```

程序执行流程：

先初始化程序
然后调用 main 包中的 main 函数
当 main 返回时，程序退出
不会等待其他非主协程完成
