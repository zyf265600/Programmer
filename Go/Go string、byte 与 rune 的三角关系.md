# Go `string`、`byte` 与 `rune` 的三角关系

### 1. 定义与本质：它们到底是什么？

| **类型**     | **别名** | **存储单元**     | **形象理解**                                        |
| ------------ | -------- | ---------------- | --------------------------------------------------- |
| **`byte`**   | `uint8`  | 1 字节 (8 bits)  | **积木块**：UTF-8 编码的最小单位。                  |
| **`rune`**   | `int32`  | 4 字节 (32 bits) | **字模**：一个完整的 Unicode 字符。**==用 ''==**    |
| **`string`** | (无)     | 只读字节序列     | **成品展示柜**：把字节按顺序封印起来。==**用 ""**== |

------

### 2. 核心冲突：字节 $\neq$ 字符 

在 ASCII 时代，1 个字节就是一个字符。但在现代 UTF-8 中：

- **英文字符**（如 `'A'`）：占 **1 个字节**。
- **中文字符**（如 `'中'`）：占 **3 个字节**。
- **Emoji 表情**（如 `'😂'`）：占 **4 个字节**。

------

### 3. `range` 遍历的“黑魔法”

**==这是 Go 编译器为了方便开发者做的一个特殊处理。当你对 `string` 进行 `range` 遍历时，它不会按字节遍历，而是自动按 `rune` 解码。==**

```go
s := "Go中国"

// 方式 A：按字节遍历 (len(s) 是 8)
for i := 0; i < len(s); i++ {
    fmt.Printf("%x ", s[i]) // 输出字节的十六进制
}

// 方式 B：按 rune 遍历 (自动解码 UTF-8)
for i, r := range s {
    fmt.Printf("%d:%c ", i, r) 
}
// 输出：0:G 1:o 2:中 5:国  <-- 注意索引从 2 直接跳到了 5！
```

**笔记关键点**：`range` 遍历字符串时，索引 `i` 是该字符**起始字节**的位置，而 `r` 是解析出来的完整 **`rune`**。

------

### 4. 强制转换的代价（搬家协议）

我们在之前的讨论中反复提到“转换意味着拷贝”，这里是具体的规则：

- **`string` $\leftrightarrow$ `[]byte`**：
  - 逻辑：为了保证 `string` 不可变，转换时必须**全文拷贝**。
  - 用途：当你需要按字节修改（如网络传输、加密）时。
- **`string` $\leftrightarrow$ `[]rune`**：
  - 逻辑：不仅是拷贝，还要进行 **UTF-8 解码/编码** 计算。
  - 用途：当你需要按字符处理（如反转字符串、统计字数、截取中文）时。

------

### 5. 什么时候用哪个？（决策树）

1. **统计长度**：
   - `len(s)`：返回的是**字节数**（常用于申请内存）。
   - `utf8.RuneCountInString(s)`：返回的是**字符数**（常用于界面显示）。
2. **反转字符串**：
   - 必须先转为 **`[]rune`**。如果转为 `[]byte`，中文字符的 3 个字节会被倒过来，变成乱码。
3. **高效拼接**：
   - 使用 `strings.Builder`，它内部操作的是 **`byte`**，因为最终生成的 `string` 就是字节流。

------

### 💡 深度总结（记在笔记末尾）：

- **`string`** 是给程序看的成品，它是**只读**的。
- **`byte`** 是给底层/网络看的，它是**最小单位**。
- **`rune`** 是给人看的，它是**逻辑字符**。
