# 01数组与切片（Go 版本）基本原理

在讨论「数组」时，必须先区分 Go 语言中的两种数据结构：

- array
- slice

在很多数据结构教材中，会区分「静态数组」与「动态数组」。

在 Go 中：

- array ≈ 静态数组
- slice ≈ 动态数组抽象

但需要注意：Go 的实现语义与 C/C++ 完全不同。

------

## Array（数组）

### Array 的本质

在 Go 中，数组是：

- 固定长度
- 值类型（value type）
- 长度是类型的一部分

定义方式：

```go
var arr [10]int
```

arr 的类型是：

```
[10]int
```

这意味着：

- arr 直接包含 10 个 int
- 它不是指针
- 它不会自动扩容

------

### 内存模型

```go
var arr [10]int
```

在内存中：

```
arr:
| int | int | int | ... | int |
```

是连续内存块。

但与 C 不同的是：

```
arr 不是一个指向首地址的指针
```

而是：

```
arr 本身就是那 10 个元素
```

------

### Array 是值类型（Value Semantics）

数组在函数传参时会整体拷贝：

```go
func f(a [10]int) {
    a[0] = 100
}

func main() {
    var arr [10]int
    f(arr)
    fmt.Println(arr[0]) // 0
}
```

说明：

```
array = value semantics
```

------

### 长度是类型的一部分

```
[10]int != [20]int
```

这是不同类型。

因此数组几乎不会在工程中直接使用。

------

## Array 的增删查改

数组固定长度，无法扩容。

### 查（Query）

```go
x := arr[3]
```

时间复杂度：

```
O(1)
```

因为：

```
address = base + index * elementSize
```

------

### 改（Update）

```go
arr[3] = 100
```

时间复杂度：

```
O(1)
```

------

### 增（Insert）

情况一：末尾追加（在未满时）

```go
arr[4] = 4
```

前提：数组尚未“逻辑满”。

时间复杂度：

```
O(1)
```

------

情况二：中间插入

需要搬移数据：

```go
for i := n; i > pos; i-- {
    arr[i] = arr[i-1]
}
arr[pos] = value
```

时间复杂度：

```
O(N)
```

------

情况三：数组已满

数组无法扩容。

必须：

1. 创建新数组
2. 拷贝数据
3. 使用新数组

```go
var newArr [20]int
copy(newArr[:], arr[:]) // [:] 表示整个区间，copy只接受slice，必须用区间操作把array转换为slice
```

时间复杂度：

```
O(N)
```

**==不能写 `newArr = arr`==**

因为：

```
[20]int != [10]int
```

类型不同，不能赋值。

------

### 删（Delete）

删除末尾：

```go
arr[n-1] = 0
```

时间复杂度：

```
O(1)
```

------

删除中间：

```go
for i := pos; i < n-1; i++ {
    arr[i] = arr[i+1]
}
```

时间复杂度：

```
O(N)
```



## Slice（切片）

在 Go 中，真正常用的是 slice。

定义：

```go
arr := make([]int, 0)
```

或：

```go
arr := []int{}
```

------

### Slice 的底层结构

Slice 本质是一个结构体：

```
slice = pointer + len + cap
```

可以抽象理解为：

```go
type slice struct {
    ptr *T
    len int
    cap int
}
```

它：

- 指向一个底层数组
- 记录当前长度
- 记录容量

------

### Slice 与 Array 的关系

```
slice 是对 array 的动态视图
```

多个 slice 可以共享同一个底层数组。

------

## Slice 的增删查改

### 查（Query）

```go
x := s[i]
```

时间复杂度：

```
O(1)
```

------

### 改（Update）

```go
s[i] = 100
```

时间复杂度：

```
O(1)
```

------

### 增（Append）

```go
s = append(s, value) // append 负责在尾部追加元素，并在容量不足时自动扩容。
```

必须接收返回值。

原因：

```
append 可能触发扩容
```

------

### 扩容机制

**==当：len < cap==**

只是：直接写入底层数组

```
s[len] = x
len++
```

没有搬移。

这是：

```
O(1)
```

**==但当：len == cap==**

触发扩容：

1. runtime 分配更大的数组
2. copy 旧数据
3. 返回新的 slice
4. 旧数组等待 GC

这是：

```
O(N)
```

但是：

扩容不是每次发生。

扩容策略是指数增长。

所以均摊下来：

```
amortized O(1)
```

------

### 中间插入

标准写法：

```go
i := 2
s = append(s, 0) // 先扩容
copy(s[i+1:], s[i:]) // func copy(dst, src []T) int
s[i] = value
```

时间复杂度：

```
O(N)
```

------

### 删除元素

删除索引 i：

```go
s = append(s[:i], s[i+1:]...) // 可变参数展开，把一个 slice 拆成多个独立参数
```

发生的本质是：

1. 你创建了两个 slice 视图
2. append 把后半段元素逐个追加到前半段

时间复杂度：

```
O(N)
```

----

### slice 能直接赋值吗？

可以。

```
s1 := []int{1,2,3}
s2 := s1
```

这是合法的。

但要理解：

```
这是浅拷贝（shallow copy）
```

发生了什么？

slice 本质是：

```
ptr + len + cap
```

赋值时：

- 复制 ptr
- 复制 len
- 复制 cap

不会复制底层数组。

内存示意：

```
底层数组:
+---+---+---+
| 1 | 2 | 3 |
+---+---+---+

s1 ----┐
       ├──→ 同一块底层数组
s2 ----┘
```

----

### copy

**函数签名**

```go
func copy(dst, src []T) int
```

含义：

- 把 `src` 中的元素复制到 `dst`
- 返回复制的元素个数

------

**复制规则**

实际复制数量：

```
min(len(dst), len(src))
```

不会越界（自动）。

------

**copy 是浅拷贝还是深拷贝？**

copy 是：元素级复制，==**是否深或浅，取决于元素本身是否包含引用**==

- 如果元素是基本类型（int, float 等） → 值复制
- 如果元素是指针 → 复制指针（不会复制指向对象）

例如：

```go
type Node struct {
    Val int
}

a := []*Node{{1},{2}}
b := make([]*Node, len(a))
copy(b, a)
```

b 和 a 指向的是同一批 Node。

------

**内存行为**

假设：

```go
s := []int{1,2,3,4,5}
```

执行：

```go
copy(s[2:], s[1:])
```

等价于：

```
dst = s[2:]
src = s[1:]
```

底层是连续内存块。

**==Go 的 copy：支持内存重叠==**

这就是为什么下面的代码安全：

```go
copy(s[i+1:], s[i:])
```

即使 src 和 dst 指向同一底层数组。

------

**时间复杂度**

copy 会执行：

```
k 次元素赋值
```

其中：

```
k = min(len(dst), len(src))
```

时间复杂度：

```
O(k)
```

最坏情况：

```
O(N)
```

------

**copy 和赋值的区别**

slice 赋值

```go
s2 := s1
```

只复制：

```
ptr + len + cap
```

O(1)

共享底层数组。

copy 操作

```go
copy(s2, s1)
```

复制：

```
所有元素
```

O(N)

==**不共享底层数组。**==

------

**copy 只能用于 slice**

所以：

```go
var arr [10]int
copy(arr, arr) // 错误
```

必须：

```go
copy(arr[:], arr[:])
```

把 array 转成 slice。

------

**典型用途**

1️⃣ 扩容时复制

```go
newS := make([]int, len(s))
copy(newS, s)
```

2️⃣ 删除元素

```go
copy(s[i:], s[i+1:])
s = s[:len(s)-1]
```

3️⃣ 插入元素

```go
s = append(s, 0)
copy(s[i+1:], s[i:])
s[i] = val
```

----

## 复杂度总结（Go 版本）

| 操作        | 时间复杂度     |
| ----------- | -------------- |
| 访问        | O(1)           |
| 修改        | O(1)           |
| 末尾 append | amortized O(1) |
| 中间插入    | O(N)           |
| 删除        | O(N)           |
| 扩容        | O(N)           |

------

## 关键语义总结（Go 思维）

在 Go 中：

```
array = 固定长度 value type
slice = 动态数组抽象
append = runtime 管理扩容
GC = 自动回收旧数组
```

数组很少直接使用。

工程与算法中几乎全部使用：

```
slice
```
